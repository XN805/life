政务服务流程数字化重构

产品助理 (实习) | 2024.12 - 2025.02

背景与痛点 (Problem)

在椿林镇便民服务中心，群众办事“填表错、排队长、重复问”是高频痛点。数据显示，仅医保核销业务的填表错误率就高达 40%。

行动与策略 (Action)
  • 用户调研：绘制用户旅程地图 (User Journey Map)，拆解出“理解政策难”和“表格填写繁”两个核心断点。
  • MVP 设计：摒弃复杂的系统开发，设计了低成本的“可视化适老交互指引”作为最小可行性产品 (MVP)。将复杂的政策转化为模块化的条件判断流程。
  • 流程重组：输出了包含 20+ 个核心节点的标准业务流程图 (Flowchart)，作为窗口人员的操作SOP。
结果 (Result)
  • 试点窗口的群众重复咨询次数下降约 30%。
  • 填表错误率显著降低，有效缓解了“二次排队”现象。

内部自动化测试效能工具

质量助理 (实习) | 2025.07 - 2025.09

背景 (Context)

在芯片及整机可靠性验证环节，原有的人工记录和分析数据方式效率低下，且容易产生人为误差。

产品化思维解决问题
  • 工具开发：利用 Python 开发自动化脚本，替代人工 Excel 处理，实现了测试数据的自动抓取与清洗。
  • 流程标准化：发现研发与测试部门对“故障”定义不一致，我主导输出了结构化的《失效分析报告》模板，统一了跨部门沟通语言。
  • 质量把控：通过模拟极端用户场景（高低温、老化），拦截了 20+ 次严重缺陷，避免了潜在的批量召回风险。
数据成果
  • 数据处理效率提升 30%，让团队能将更多精力投入到核心问题分析中。

丹江流域水质数字孪生平台

项目负责人 (PM Owner)

需求分析

监管人员面对海量的 PH 值、重金属含量等 excel 表格数据，无法快速直观地定位污染源头，决策滞后。

产品方案
  • 可视化交互设计：设计了“3D全景漫游 + 悬浮数据看板”的界面形态。用户既可以宏观查看流域全貌，也可以点击监测点“下钻”查看实时数据。
  • 技术标准制定：为了确保前端展示与后端数据的实时性，我主导撰写了接口文档，定义了 JSON 数据格式和异常报警阈值。
价值
  • 实现了从“被动查表”到“主动预警”的管理模式转变。
  • 通过标准化接口文档，降低了前后端开发人员的沟通成本。

AI 医疗诊断模型优化策略

算法与数据研究

产品视角

AI 产品的核心壁垒不仅在于算法模型，更在于高质量的数据治理和持续的反馈闭环。

执行策略
  • 数据资产治理 (ETL)：面对原始临床数据“脏、乱、差”的问题，制定了清洗与标注的 SOP，为模型训练提供高质量“燃料”。
  • Bad Case 闭环：建立误判案例库。不只是统计准确率，而是深入分析每一个错误案例（如图像模糊、特征重叠），并将分析结果反馈给算法参数调整，完成了“数据-模型-评估-优化”的闭环。
